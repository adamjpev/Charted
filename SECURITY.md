# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Charted, please report it privately — do not open a public GitHub issue.

- Email: **security@getcharted.net** (fallback: contact@getcharted.net)
- Include: a description of the vulnerability, steps to reproduce, and any relevant logs, PoC code, or screenshots.

We aim to acknowledge reports within 3 business days and to provide a resolution timeline after initial triage.

## Scope

This policy covers the `charted-server` backend, including authentication, the REST/Socket.IO APIs, and infrastructure code in this repository.

## Supported Versions

Only the latest version deployed on `main` is supported with security fixes.

## Disclosure Policy

Please give us a reasonable amount of time to investigate and address a report before any public disclosure. We will credit reporters who wish to be acknowledged once a fix has shipped.
